# Pita Bros Supply OS — v0 backend

Minimal runnable FastAPI scaffold for the Captain Submit + Manager
Dispatch apps. Pilot location: Wola. Pilot supplier: Pago.

## What's here

- `app/main.py` — FastAPI app with API routes.
- `app/models.py` — Pydantic models matching
  `../docs/pita-supply-os-v1/DATA_MODEL.md`.
- `app/seed_loader.py` — reads seed CSVs from
  `../docs/pita-supply-os-v1/seed/` for local dev.
- `app/sheets.py` — Google Sheets adapter stub (production data backend).
- `app/suggestion.py` — the suggestion engine
  (`target - current → ceil(... / units_per_purchase_unit)`).
- `tests/test_suggestion.py` — unit tests for the engine.

## Run locally

```sh
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
cp .env.example .env
uvicorn app.main:app --reload --port 8000
```

Open in your browser:

- http://localhost:8000/health
- http://localhost:8000/api/products
- http://localhost:8000/api/captain/orderable?location_id=WOLA&supplier_id=SUP_PAGO

## Tests

```sh
pytest
```

## Deploy to Railway

See `../docs/pita-supply-os-v1/BUILD_PLAN.md` step 6. Short version:

```sh
railway init
railway up
```

Set env vars in the Railway dashboard. The most important:
`SUPPLY_OS_DATA_BACKEND=sheet`, `GOOGLE_SHEET_ID`,
`GOOGLE_SERVICE_ACCOUNT_JSON`, `POSTHOG_API_KEY`.
