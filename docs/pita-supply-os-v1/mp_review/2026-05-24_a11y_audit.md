# MP Captain UI — WCAG 2.1 AA Audit

Source: Magic Patterns generated zip (id `ea09d61c-8332-47a4-b619-4c1bb324afec`)
Reviewer: general-purpose sub-agent (Claude)
Target users: kitchen workers on mobile phones, some non-native Polish speakers, in noisy/distracted kitchen environments

## BLOCKER

### B1 — 4.1.2 Name, Role, Value: AuthModal lacks dialog semantics
**File:** `AuthModal.tsx:14-16`
The modal is a plain `<div>` overlay — no `role="dialog"`, no `aria-modal="true"`, no `aria-labelledby`, no focus trap, no Esc handler, no body scroll lock.
**Fix:**
```tsx
<div role="dialog" aria-modal="true" aria-labelledby="auth-title" className="fixed inset-0 …">
  <div className="bg-white …">
    <h2 id="auth-title" …>Wpisz kod miejsca</h2>
```
Add `useEffect` to focus the input, trap Tab/Shift-Tab, close on Escape, set `document.body.style.overflow = 'hidden'`. **NOTE for integration**: replaced entirely by our `AuthGate` which already handles these correctly.

### B2 — 4.1.3 Status Messages: Toast not announced
**File:** `Toast.tsx:18-29`
No `role="status"` / `role="alert"` and no `aria-live`. Toast is the only confirmation of submit success or failure.
**Fix:**
```tsx
<div
  role={isSuccess ? 'status' : 'alert'}
  aria-live={isSuccess ? 'polite' : 'assertive'}
  aria-atomic="true"
>
```
Also add a visible close button — 3s auto-dismiss is too short for slower readers (WCAG 2.2.1 Timing Adjustable).

### B3 — 1.3.1 Info & Relationships / 3.3.2 Labels: ProductCard inputs have no `htmlFor`
**File:** `ProductCard.tsx:94-110, 130-146`
Both `<label>` elements are not bound to their `<input>`.
**Fix:**
```tsx
<label htmlFor={`current-${item.product_id}`}>Obecny stan</label>
<input id={`current-${item.product_id}`} … aria-describedby={`unit-${item.product_id}`} />
<span id={`unit-${item.product_id}`}>{item.inventory_unit}</span>
```
Repeat for "Zamawiasz" (`final-${product_id}`). Apply the same fix in `ReasonPicker.tsx:21-37` and `42-50`.

### B4 — 1.3.1 / 3.3.1 Error Identification: Reason-required state has no programmatic association
**Files:** `ProductCard.tsx:156-161`, `ReasonPicker.tsx`
When `requiresReason` is true, the pill (`{message}`) and the empty `<select>` are visually adjacent but not linked. Nothing tells AT that the select is in an error state.
**Fix:** On the select add `aria-invalid={requiresReason && !value}` and `aria-describedby="reason-error-…"`. Add `required` on textarea when `value === 'OTHER'`. Status pill should be `role="status"`.

## HIGH

### H1 — 2.5.5 Target Size: Header menu button & Sticky bar buttons too small
**Files:** `Header.tsx:17-22`, `StickyActionBar.tsx`, `SupplierPicker.tsx:32`
Menu button ~32px, Sticky bar buttons ~40px (py-2.5), Supplier chips ~36px (py-2). All under 44px.
**Fix:** Header menu `p-2.5` (44px). Sticky bar `py-3 px-6`. Supplier chips `py-3`.

### H2 — 1.4.3 Contrast: Gray-400 text fails AA on gray-50 backgrounds
**Files:**
- `ProductCard.tsx:107, 143` `text-gray-400` (inventory unit suffix) on `bg-gray-50` ≈ 2.85:1 — **FAILS 4.5:1**
- `ProductCard.tsx:122` `text-gray-400 italic` ("brakuje X szt") — same failure, ~2.85:1
- `ProductCard.tsx:115` `text-gray-500` (Sugestia label) on `bg-gray-50` ≈ 4.3:1 — borderline fail
- `SupplierPicker.tsx:33` `text-gray-500` on `bg-gray-100` (sent state) ≈ 4.1:1 — fails

**Fix:** Replace every `text-gray-400` used for content with `text-gray-600` (7.2:1). Replace `text-gray-500` on gray-50 surfaces with `text-gray-700`.

### H3 — 1.4.11 Non-text contrast: Left border colors fail 3:1
**File:** `ProductCard.tsx` STATE_COLORS
`border-l-yellow-400` (#FACC15 on white ≈ 1.7:1) — **FAILS 3:1**
`border-l-green-500` (#22C55E ≈ 1.96:1) — **FAILS**
`border-l-orange-500` (#F97316 ≈ 2.7:1) — **FAILS**
`border-l-red-500` (#EF4444 ≈ 3.8:1) — passes
**Fix:** `border-l-yellow-700`, `border-l-green-700`, `border-l-orange-600`. Keep red-500.

### H4 — 1.4.1 Use of Color: State conveyed only by color
**Files:** `ProductCard.tsx:18-43, 150-154`
A colorblind user can't distinguish 4-state vocabulary.
**Fix:** Add lucide-react icon prefix to pill: `✓` green, `Info` yellow, `AlertTriangle` orange, `AlertOctagon` red.

### H5 — 4.1.2 / 1.1.1: Header emojis have no text labels
**File:** `Header.tsx:25-35`
`<span>📍</span> {locationName}` — emoji is announced as "round pushpin" with no "Location:" prefix.
**Fix:**
```tsx
<span aria-hidden="true">📍</span>
<span className="sr-only">Lokalizacja: </span>
{locationName || 'Ładowanie...'}
```

### H6 — 3.2.2 / 3.3.4: `window.confirm` for draft restore
**File:** `Captain.tsx:70-81`
Native confirm blocks main thread, English-only buttons, English prompt.
**Fix:** Replace with in-app dialog (reuse modal pattern), Polish copy: "Wznowić szkic z godziny HH:MM?" / "Wznów" / "Odrzuć".
